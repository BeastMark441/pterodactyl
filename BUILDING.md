# Локальная разработка
Pterodactyl теперь работает на React, TypeScript и Tailwindcss, используя webpack в качестве основы для генерации скомпилированных ассетов.
Релизные версии Pterodactyl включают предварительно скомпилированные, минифицированные и хешированные ассеты, готовые к использованию.

Однако, если вы хотите использовать собственные темы или вносить изменения в React-файлы, вам потребуется система сборки для генерации этих скомпилированных ассетов. Для настройки окружения вам как минимум понадобятся:

* [Node.js](https://nodejs.org/en/) v14.x.x
* [Yarn](https://classic.yarnpkg.com/lang/en/) v1.x.x
* [Go](https://golang.org/) 1.17.x

### Установка зависимостей
```bash
yarn install
```

Команда выше загрузит все зависимости, необходимые для сборки ассетов Pterodactyl. После этого останется лишь выполнить команду ниже, чтобы генерировать ассеты во время разработки. 
Пока вы не выполните эту команду хотя бы один раз, на Panel, скорее всего, будет возникать ошибка 500 из-за отсутствующего файла `manifest.json`. Этот файл создаётся командами ниже.

```bash
# Собрать скомпилированный набор ассетов для разработки.
yarn run build

# Автоматически пересобирать ассеты при их изменении. Это позволяет
# обновлять страницу и сразу видеть изменения.
yarn run watch
```

### Горячая перезагрузка модулей
Для более опытных пользователей также поддерживается Hot Module Reloading, позволяющий быстро видеть изменения, которые вы вносите в шаблоны Vue, без необходимости перезагружать текущую страницу. Чтобы начать, достаточно выполнить команду ниже.

```bash
PUBLIC_PATH=http://192.168.1.1:8080 yarn run serve --host 192.168.1.1
```

В этой команде есть две _очень важные_ части, на которые нужно обратить внимание и изменить под своё окружение. Первая — это флаг `--host`, он обязателен и должен указывать на машину, где будет запущен сервер `webpack-serve`. Вторая — это переменная окружения `PUBLIC_PATH`, которая задаёт URL HMR-сервера и добавляется ко всем URL ассетов, используемых в Pterodactyl.


#### Окружение разработки
Если вы используете окружения [`pterodactyl/development`](https://github.com/pterodactyl/development), что настоятельно рекомендуется, вы можете просто выполнить `yarn run serve`, чтобы запустить HMR-сервер — дополнительная настройка не требуется.

### Сборка для production
Когда ваши файлы готовы и их нужно выкладывать на рабочий сервер, потребуется сгенерировать скомпилированные, минифицированные и хешированные ассеты для продакшена. Для этого выполните команду ниже:

```bash
yarn run build:production
```

Это создаст production JS-бандл и связанные с ним ассеты, все они будут находиться в `public/assets/`, и их нужно будет загрузить на ваш сервер или CDN для использования клиентами.

### Запуск Wings
Чтобы запускать `wings` в режиме разработки, достаточно обычным образом настроить конфигурационный файл при добавлении нового узла, а затем собрать и запустить локальную версию Wings, выполнив `make debug` в каталоге с кодом Wings. Это необходимо запускать в какой-либо Linux VM, локально на macOS или Windows это не работает.
